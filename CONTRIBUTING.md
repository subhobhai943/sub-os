# Contributing to SUB OS

Thank you for your interest in SUB OS! This is currently a **365-day solo challenge** to build an operating system from scratch.

## Current Status

This project is in active development as part of a one-year challenge. While this is primarily a solo endeavor, feedback and suggestions are appreciated!

## How You Can Help

### 1. Provide Feedback
- Open issues for bugs or suggestions
- Share ideas for optimization
- Suggest resources or documentation improvements

### 2. Code Review
- Review pull requests (if any)
- Suggest improvements to code structure
- Point out potential issues or bugs

### 3. Documentation
- Help improve README or other docs
- Fix typos or unclear explanations
- Add comments to complex code sections

### 4. Testing
- Test SUB OS on different hardware
- Report compatibility issues
- Share performance benchmarks

## Development Guidelines

### Code Style
- **C Code**: Follow K&R style with 4-space indentation
- **Assembly**: Use meaningful labels and comments
- **Comments**: Explain *why*, not just *what*

### Commit Messages
Format: `type(scope): message`

Types:
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation
- `refactor`: Code refactoring
- `test`: Testing
- `chore`: Maintenance

Example: `feat(memory): implement page allocator`

### Testing
- Test all changes in QEMU before committing
- Verify boot process works correctly
- Document any hardware-specific issues

## Questions?

Feel free to open a discussion or issue on GitHub!
